================================================================================
                         DATABASE INTEGRATION
================================================================================

All marks are stored in SUPABASE POSTGRESQL:

Table: marks
┌──────────────────────────────────────────────────────────────┐
│ Field              │ Type       │ Description                │
├──────────────────────────────────────────────────────────────┤
│ id                 │ BIGSERIAL  │ Primary Key (Auto)        │
│ student_id         │ BIGINT     │ Foreign Key → students    │
│ subject            │ VARCHAR    │ Subject Name              │
│ marks_obtained     │ DECIMAL    │ Marks 0-100              │
│ grade              │ CHAR(2)    │ A+, A, B, C, D, F        │
│ homework_status    │ VARCHAR    │ Complete/Incomplete       │
│ teacher_comments   │ TEXT       │ Optional feedback         │
│ created_at         │ TIMESTAMP  │ Auto (Record created)     │
│ updated_at         │ TIMESTAMP  │ Auto (Last modified)      │
└──────────────────────────────────────────────────────────────┘

Features:
  ✅ Auto-timestamps
  ✅ Cascading deletes (delete student = delete marks)
  ✅ Proper indexing for performance
  ✅ Real-time synchronization

================================================================================
                            API ENDPOINTS
================================================================================

GET    /api/marks
  Gets all marks or filter by studentId
  Query: ?studentId=5
  Returns: Array of marks objects

POST   /api/marks
  Creates new marks entry
  Body: { student_id, subject, marks_obtained, homework_status, ... }
  Returns: Created marks object

PUT    /api/marks/[id]
  Updates existing marks entry
  Body: { subject, marks_obtained, homework_status, ... }
  Returns: Updated marks object

DELETE /api/marks/[id]
  Deletes marks entry
  Returns: { success: true }

GET    /api/students
  Gets all students (for selection dropdown)
  Returns: Array of student objects
